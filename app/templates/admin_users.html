Authentication (OAuth / SSO)

Optional sign-in alongside local login. Google/GitHub OAuth: set a provider's client id + secret; callback URL is <site_base_url>/auth/oauth/<provider>/callback. With oauth_auto_provision off, SSO only logs in users whose email matches an existing account. Set local_login_enabled=false to hide the password form once SSO works.

🔒 Tailscale sign-in: set tailscale_auth_enabled=true to trust Tailscale Serve identity headers. This is only safe when the app is reachable only via Serve on localhost (drop the tailnet-IP port bind), and you list the proxy address in tailscale_trusted_proxies. ADSBuddy currently sees your connection coming from {{ client_host }} — if you're connected via Serve, that's the value to add (e.g. {{ client_host }}/32). Blank = Tailscale header auth is refused.
